## 🔍 Exploratory Testing

### Why We Need It

**Business:**
- Uncovers defects that scripted tests miss — real users don't follow scripts
- Reduces the cost of late-stage bug discovery by finding edge cases early
- Provides rapid feedback during time-constrained releases with minimal prep
- Builds confidence that the product behaves correctly in unexpected scenarios

**Development Team:**
- Gives testers freedom to investigate areas that feel risky or unstable
- Surfaces usability and logic issues that formal test cases rarely catch
- Complements automated and manual regression — covers the unknown unknowns
- Generates new test ideas that feed back into the formal test suite

---

### Aim
- Investigate the application without predefined steps to find unexpected behavior
- Learn the system deeply by interacting with it as a real user would
- Document findings in real time to ensure nothing is lost between sessions
- Use heuristics and intuition to target high-risk and newly changed areas

---

### Deliverables
- Session-based test charters (goal, scope, and duration per session)
- Bug reports and observations logged during each session
- Coverage notes (areas explored, areas skipped, and why)
- Debrief summary with findings, risks identified, and follow-up actions
